## Requirement
- REQ-39

## Evidence
- `src/keyring_store.rs`: single unified implementation using `keyring::Entry` — no `#[cfg(target_os)]` guards.
- `Cargo.toml` platform-specific features:
  - macOS: `keyring = { version = "3", features = ["apple-native"] }` (Security framework)
  - Linux: `keyring = { version = "3", features = ["linux-native"] }` (kernel keyutils, pure Rust)
  - Windows: `keyring = { version = "3", features = ["windows-native"] }` (Credential Manager)
- `keyring set|get|delete` commands wired in CLI dispatch (`src/commands/keyring.rs`).
- `GITVAULT_KEYRING=1` identity loading path exercised through unit and integration coverage.
- Smoke test `keyring_set_get_delete_roundtrip` tolerates CI environments without a keyring daemon.

## In-Progress Extension Scope
- Add optional keyring passphrase fetch for identity loading with `--no-prompt` compatibility.
- Add optional SSH-agent source after keyring in identity precedence.
- Enforce fail-closed behavior for zero/multiple usable agent keys unless explicitly selected.
- Apply identity-source behavior across encrypt, decrypt, materialize, run, check, and recipient/rotate paths that resolve local identity.
