## Requirement
- REQ-39

## Evidence

### AC1, AC2 — Keyring save/status/delete, native platform backends
- `src/keyring_store.rs`: single unified implementation using `keyring::Entry` — no `#[cfg(target_os)]` guards.
- `Cargo.toml` platform-specific features:
  - macOS: `keyring = { version = "3", features = ["apple-native"] }` (Security framework)
  - Linux: `keyring = { version = "3", features = ["linux-native"] }` (kernel keyutils, pure Rust)
  - Windows: `keyring = { version = "3", features = ["windows-native"] }` (Credential Manager)
- `keyring set|get|delete` commands wired in CLI dispatch (`src/commands/keyring.rs`).
- Smoke test `keyring_set_get_delete_roundtrip` tolerates CI environments without a keyring daemon.

### AC3 — Optional keyring passphrase fetch for identity loading
- `src/keyring_store.rs`: `keyring_get_identity_passphrase` and `keyring_set_identity_passphrase` store/fetch SSH key passphrases under `{account}-passphrase` account key, separate from the identity key itself.
- `src/identity.rs`: `try_fetch_ssh_passphrase(service, account, no_prompt)` checks `GITVAULT_IDENTITY_PASSPHRASE` env var first, then OS keyring. In CI (`no_prompt=true`) only env var is tried.
- `src/crypto.rs`: `ZeroizingPassphrase` implements `age::Callbacks` for `request_passphrase`; `parse_identity_any_with_passphrase` wraps SSH identity with passphrase callback so passphrase-encrypted SSH keys decrypt silently.
- CLI: `gitvault keyring set-passphrase`, `get-passphrase`, `delete-passphrase` commands let users manage stored passphrases.

### AC4, AC5 — SSH-agent fallback and identity selector
- OS keyring is tried automatically as step 3 in the identity resolution chain; no env var required.
- SSH-agent fallback is step 4: `list_ssh_agent_keys()` → single age-compatible key → `find_ssh_key_file` to locate the private key file.
- `--identity-selector <value>` and `GITVAULT_IDENTITY_SELECTOR` allow explicit key disambiguation.

### AC6 — SSH-agent key as age-compatible identity material
- `src/identity.rs` `find_ssh_key_file`: 3-priority scan: (1) standard names (`id_ed25519`, `id_ecdsa`, `identity`), (2) filename matches SSH-agent comment, (3) full `~/.ssh/` directory scan sorted deterministically, skipping `.pub`/config/known_hosts files. Uses `ssh-keygen -l -E sha256` fingerprint match.
- `src/identity.rs` `probe_identity_sources`: SSH-agent "1 key found" branch now calls `find_ssh_key_file`; reports `SourceNotAvailable` with actionable message when the private key file cannot be located on disk.
- Fail-closed: zero or multiple usable agent keys → `source-not-available` unless `--identity-selector` selects one explicitly.
