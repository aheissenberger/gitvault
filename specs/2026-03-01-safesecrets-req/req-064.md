## Verification
- Integration tests for Husky adapter discovery and deterministic missing-adapter outcomes.

## Implementation-facing Acceptance Scenarios
- R064-S1: Given Husky integration is configured and adapter is on PATH, when check/harden runs, then adapter discovery succeeds deterministically.
- R064-S2: Given adapter is absent and CI/`--no-prompt` is active, when command runs, then process fails with stable exit code and actionable install message.
- R064-S3: Given staged plaintext secret material, when Husky pre-commit path executes gitvault checks, then commit is rejected.
- R064-S4: Given drift exists, when Husky pre-push path executes gitvault checks, then push is rejected.
- R064-S5: Given adapter is installed after initial core-only deployment, when command re-runs, then integration works without recompiling core.
