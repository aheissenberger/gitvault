## Evidence
- `src/ssm.rs` `cmd_ssm_pull`: fetches all parameters under `/{app}/{env}/` via paginated `get_parameters_by_path`, writes refs to `.ssm-refs.json`.
- `src/ssm.rs` `cmd_ssm_diff`: compares local refs vs. live SSM keys, prints `+` / `-` lines with masked values.
- `src/ssm.rs` `cmd_ssm_set`: writes a single `SecureString` parameter to SSM then updates local refs.
- `src/ssm.rs` `cmd_ssm_push`: reads all local refs, looks up each key in environment, pushes to SSM.
- `src/cli.rs` `SsmAction` enum: `Pull`, `Diff { reveal }`, `Set { key, value }`, `Push` — all wired in dispatch.
- Unit tests in `ssm::tests` cover path formatting, refs serialisation, and the empty-file fallback.
