## Verification
- Integration tests for `lefthook` adapter discovery and idempotent repeated-run behavior.
- Regression tests preserving plaintext and drift enforcement guarantees.

## Implementation-facing Acceptance Scenarios
- R066-S1: Given effective gitvault config selects `lefthook` and adapter exists, when integration check runs, then adapter is selected deterministically.
- R066-S2: Given adapter is missing and CI/`--no-prompt` mode is active, when command runs, then deterministic failure with install guidance is returned.
- R066-S3: Given staged plaintext secret material, when lefthook pre-commit executes gitvault checks, then commit is rejected.
- R066-S4: Given repository drift condition, when lefthook pre-push executes gitvault checks, then push is rejected.
- R066-S5: Given repeated harden/status runs with unchanged config, when commands execute repeatedly, then results remain idempotent and stable.
