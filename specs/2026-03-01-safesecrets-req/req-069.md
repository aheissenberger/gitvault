## Requirement
- REQ-69

## Context
Users with low-frequency Copilot integration needs want deterministic, first-party command output for Skills and project context without standing up an MCP server or custom VS Code extension.

## Goal
Provide a minimal `gitvault ai` command group that exposes stable print primitives for AI tooling bootstrap and automation.

## Command Surface
- `gitvault ai skill print`
- `gitvault ai context print`

## Output Contract
- Human mode prints plain text/markdown suitable for direct copy or redirection.
- JSON mode emits a stable machine envelope with top-level protocol/tool/success markers and structured payload/error fields.
- Commands remain deterministic across CI, local terminal, and agent execution.

## Failure Handling
- Content is embedded at compile time; missing source files cause a build error, not a runtime error.
- Invalid serialized output in JSON mode fails with parse-safe diagnostics.
- Commands are always deterministic; no runtime file I/O is performed.

## Verification
- Unit and integration tests cover human and JSON output paths for both commands.
- Tests validate deterministic schema keys for JSON envelope.
- `cargo xtask cli-help` is re-run when command/help surface changes and `docs/ai/cli-help.json` is updated accordingly.

## Implementation-facing Acceptance Scenarios
- R069-S1: Given a built binary, when `gitvault ai skill print` runs, then canonical skill content (from `docs/ai/skill.md` embedded at compile time) is emitted.
- R069-S2: Given a built binary, when `gitvault ai context print` runs, then canonical context content (from `docs/ai/AGENT_START.md` embedded at compile time) is emitted.
- R069-S3: Given `--json`, when either command runs successfully, then output conforms to the stable envelope schema.
- R069-S4: Commands never fail at runtime due to missing files — content is always present in the binary.
- R069-S5: Given `--no-prompt`, when either command runs, then no interactive prompts are attempted.
