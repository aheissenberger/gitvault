## Requirement
- REQ-67

## Context
REQ-64, REQ-65 and REQ-66 introduce external hook-manager adapters for Husky, pre-commit and lefthook. The adapter selection must be declared explicitly by the repository owner using gitvault's own canonical config file rather than reading from hook-manager config files.

## Config File: `.gitvault/config.toml`

```toml
[hooks]
adapter = "husky"   # or "pre-commit" or "lefthook"
```

Supported `adapter` values: `husky`, `pre-commit`, `lefthook`. Any other non-empty value is a `Usage` error. Absent key or missing file → built-in default adapter.

The adapter binary discovered on PATH is `gitvault-<adapter>` (e.g. `gitvault-husky`).

## Verification
- Unit tests: `.gitvault/config.toml` absent → built-in adapter, no error.
- Unit tests: `[hooks] adapter = "pre-commit"` → activation attempted.
- Unit tests: `[hooks]` section absent → built-in adapter, no error.
- Unit tests: `adapter = "unknown"` → `GitvaultError::Usage`.
- Integration tests: `gitvault harden` without config file → only built-in hooks installed.

## Implementation-facing Acceptance Scenarios
- R067-S1: Given `.gitvault/config.toml` is absent, when `harden` runs, then only built-in git hooks are installed and no external-adapter logic runs.
- R067-S2: Given `.gitvault/config.toml` has `[hooks] adapter = "pre-commit"` and `gitvault-pre-commit` is on PATH, when `harden` runs, then the pre-commit adapter is invoked.
- R067-S3: Given `.gitvault/config.toml` exists but has no `[hooks]` section, when `harden` runs, then only built-in git hooks are installed.
- R067-S4: Given `[hooks] adapter = "lefthook"` but `gitvault-lefthook` is absent from PATH and `--no-prompt` is set, when `harden` runs, then the command exits non-zero with an actionable install message.
- R067-S5: Given `[hooks] adapter = "unknown-tool"`, when `harden` runs, then a `Usage` error is returned immediately.
