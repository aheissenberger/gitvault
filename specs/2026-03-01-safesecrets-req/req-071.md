## Requirement

REQ-71 — `gitvault init` onboarding command

### Rationale

The current first-run experience requires four separate commands in the right order:

```bash
gitvault identity create
gitvault recipient add <pubkey>
gitvault harden
# manually create .gitvault/config.toml if wanted
```

A new developer on a project that already uses gitvault must also figure out:
- That they need an age identity before anything else
- How to share their public key with a teammate
- That `harden` needs to run before the hooks fire

`gitvault init` collapses this into a single guided entry point while remaining fully
non-interactive for CI bootstrap and scripted environment setup.

### CLI design

```
# Interactive (new developer, first time):
gitvault init

# Non-interactive / CI (all decisions use defaults):
gitvault init --no-prompt

# Set environment during init:
gitvault init --env staging

# Export identity to file in addition to keyring:
gitvault init --out ~/.age/id.key

# Full CI bootstrap with explicit identity file:
GITVAULT_IDENTITY=~/.age/id.key gitvault init --no-prompt
```

### Step flow

```
gitvault init
│
├─ Step 1: Identity
│   ├─ Resolve existing identity source  →  ✓ found (source reported), skip creation
│   └─ No identity found
│       ├─ [interactive]  Create new? [Y/n] → gitvault identity create
│       └─ [--no-prompt]  Create automatically (classic profile, keyring)
│
├─ Step 2: Recipients
│   ├─ .secrets/recipients exists + non-empty  →  ✓ already present, skip
│   └─ Missing / empty
│       ├─ Auto-add own public key (from step 1 identity)
│       └─ [interactive]  Add more recipients? (paste pubkeys, blank line to finish)
│
├─ Step 3: Harden  →  delegates to existing cmd_harden
│
└─ Step 4: Config
    ├─ .gitvault/config.toml exists  →  ✓ already present, skip
    └─ Create empty file with comment header
```

```json
    { "step": "recipients",  "status": "created",        "count": 1 },
    { "step": "harden",      "status": "already_present" },
    { "step": "config",      "status": "created" }

    "gitvault encrypt .env",
    "gitvault status"
  ]

```

### `--no-prompt` automatic defaults

| Decision | Default |
|----------|---------|
| Identity profile | `classic` |
| Identity storage | OS keyring (fail if unavailable unless `--out` given) |
| Recipients | Own public key only |
| Environment | `dev` (or `--env` flag value) |
| Config file | Created if absent, empty with comment header |
