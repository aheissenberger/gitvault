## Verification
- Integration tests for `pre-commit` adapter discovery and deterministic non-interactive error contracts.
- Regression tests that preserve plaintext and drift checks across manager invocation paths.

## Implementation-facing Acceptance Scenarios
- R065-S1: Given `.pre-commit-config.yaml` integrates gitvault and adapter is available, when integration check runs, then discovery is deterministic.
- R065-S2: Given adapter/runtime prerequisites are missing in CI/`--no-prompt`, when command runs, then failure is machine-parseable and actionable.
- R065-S3: Given staged plaintext secret material, when `pre-commit` hook invokes gitvault checks, then commit is blocked.
- R065-S4: Given drift exists, when pre-push path in manager chain invokes gitvault checks, then push is blocked.
- R065-S5: Given user-maintained `pre-commit` configuration, when integration validation runs, then no destructive silent rewrite occurs.
