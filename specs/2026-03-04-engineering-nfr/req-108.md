# REQ-108: Contextual File Read Utility

**ID:** S-20260304-R0108
**Status:** Partial (utility implemented; gradual adoption per NFR-002)
**Priority:** Low
**Category:** Engineering / Diagnostics

## Problem

`std::fs::read_to_string` appears ~76 times in the codebase. Each call site produces its
own error message (or none), leading to inconsistent diagnostics — e.g., "No such file or
directory" without the file path, making bugs hard to diagnose in the field.

## Requirement

A `fs_util::read_text(path: &Path) -> Result<String, GitvaultError>` utility SHALL provide
a canonical wrapper that includes the file path in the error message. New call sites MUST
use `fs_util::read_text`. Existing call sites SHOULD be migrated incrementally under NFR-002.

## Acceptance Criteria

- AC1: `fs_util::read_text` exists and includes the file path in any error message.
- AC2: New code uses `fs_util::read_text` instead of `std::fs::read_to_string` directly.
- AC3: Migration is tracked under NFR-002.

## Implementation

- `src/fs_util.rs`: `pub fn read_text`
