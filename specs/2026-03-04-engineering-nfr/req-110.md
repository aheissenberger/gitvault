# REQ-110: Auto-discover Encrypted Fields in Structured Files

**ID:** S-20260304-R110
**Status:** Implemented
**Priority:** Medium
**Category:** Developer Experience / Usability

## Problem

`gitvault decrypt package.json --reveal` (or any structured config file where only *some*
values are age-encrypted) fails with the opaque error:

```
Error: Decryption error: Failed to create decryptor: Header is invalid
```

The root cause: when `--fields` is omitted the command falls through to the whole-file
binary age-stream path (`crypto::decrypt_stream`), which immediately rejects valid JSON/YAML/TOML
because those files are not age-binary streams. The error message gives the developer no
actionable information.

The workaround — `--fields api_key,db_password` — requires the developer to know which
fields are encrypted ahead of time, defeating the point of a "reveal" command.

## Decision

**Auto-discover** encrypted fields when `--fields` is omitted and the input file is a
recognised structured format (JSON, YAML, TOML). Walk every string value in the document,
identify those that begin with the age ASCII-armor header (`-----BEGIN AGE ENCRYPTED FILE-----`),
collect their dot-path keys, and decrypt only those fields.

The existing `--fields` path, `--value-only` path, and `.age` whole-file stream path are
**unchanged**.

## New Public API (`src/structured`)

```rust
/// Recursively collect dot-path keys of every string value in the document
/// that is age ASCII-armor encrypted.
///
/// Returns an empty `Vec` when no encrypted values are found.
///
/// # Errors
/// Returns [`GitvaultError`] if the content cannot be parsed for the given extension.
pub fn collect_encrypted_field_paths(
    content: &str,
    ext: &str,
) -> Result<Vec<String>, GitvaultError>;

/// Decrypt the listed fields in the document in-memory and return the result as a `String`.
///
/// When `skip_undecryptable` is `true`, fields that fail decryption emit a warning to stderr
/// and are left as-is; the call still succeeds for all other fields.
///
/// # Errors
/// Returns [`GitvaultError`] on parse errors or (when `skip_undecryptable` is `false`)
/// on any field decryption failure.
pub fn decrypt_fields_content(
    content: &str,
    ext: &str,
    fields: &[&str],
    identity: &dyn age::Identity,
    skip_undecryptable: bool,
) -> Result<String, GitvaultError>;
```

## Command Flow Change (`src/commands/decrypt.rs`)

```
decrypt <FILE> [options]
├── --value-only              → existing .env value-only path (unchanged)
├── --fields given            → existing explicit field-decrypt path (unchanged)
├── extension is .age         → existing whole-file binary age-stream path (unchanged)
└── extension is .json/.yaml/.yml/.toml AND no --fields
    ├── collect_encrypted_field_paths(content, ext)
    ├── if empty → print notice, return Ok
    └── decrypt_fields_content(content, ext, paths, identity, skip_undecryptable=true)
        ├── --reveal  → print String to stdout
        └── else      → atomic_write to resolved output path
```

## Design Choices

| Choice | Rationale |
|---|---|
| Recursive walker (not dot-path navigator) | `json_field_mut` only traverses an explicit path list; auto-discover needs to visit every leaf |
| `skip_undecryptable = true` for auto-discover | The identity may be a recipient for only some fields (e.g., shared config with per-team secrets) |
| Warnings to stderr | stdout is reserved for decrypted content when `--reveal` is active |
| Extension guard (not content-sniff) | Deterministic, fast, consistent with `encrypt --fields` extension guard |
| `decrypt_fields` refactored to call `decrypt_fields_content` | Eliminates logic duplication; all file-format logic lives in one place |

## Test Plan

- Unit: `collect_encrypted_field_paths` finds nested keys in all three formats.
- Unit: returns empty vec for fully-plain documents.
- Unit: `decrypt_fields_content` with `skip_undecryptable=true` leaves undecryptable field intact.
- Unit: `decrypt_fields` (file-write) still passes all existing tests.
- Integration: `cmd_decrypt` without `--fields` on a partially-encrypted JSON file decrypts
  all encrypted fields and leaves plain fields untouched.
