## Overview

Two SSH binaries (`ssh-add`, `ssh-keygen`) are called from `identity.rs` to list agent keys and verify fingerprints. Before this requirement, each call site was an inline `Command::new` block with no consistent suppression of interactive password prompts. This is the same class of problem that motivated `src/git.rs` (REQ-90).

## `src/ssh.rs` API

```rust
/// List ED25519 keys in the SSH agent.
/// Returns raw stdout bytes on success; callers parse the key lines.
pub fn ssh_add_list_keys() -> Result<Vec<u8>, SshError>;

/// Get the SHA-256 fingerprint line for a key file.
/// Returns `None` if ssh-keygen cannot be run or the file is invalid.
pub fn ssh_keygen_fingerprint(path: &Path) -> Option<String>;
```

## Security Defaults

`SSH_ASKPASS_REQUIRE=never` is set on every call so that gitvault never blocks waiting for interactive SSH passphrase input in CI or non-terminal contexts. Without this, `ssh-keygen -l` on an encrypted key file would hang waiting for a passphrase on some systems.

## Migration

`identity.rs::list_ssh_agent_keys` and `identity.rs::find_ssh_key_file` replace their inline `Command::new` blocks with calls to the new wrapper functions.
