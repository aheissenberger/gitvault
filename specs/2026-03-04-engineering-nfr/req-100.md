## Overview

`plugin.rs` contains a hand-rolled `which_binary()` function that searches `PATH` for adapter binaries. While it handles `PATHEXT` on Windows, it misses:

- UNC paths (`\\server\share\bin`)
- Binary names with spaces that require quoting
- Symlink resolution edge cases

The `which` crate handles all of these correctly and is the accepted standard for binary resolution in the Rust ecosystem.

## Change

Replace the private `which_binary()` implementation and its callers with `which::which()`:

```rust
// Before
fn which_binary(name: &str) -> Result<PathBuf, ()> { ... }  // 30+ lines

// After
use which::which;
// find_adapter_binary calls which::which(binary) directly
```

## Crate Justification

| Criterion | Result |
|-----------|--------|
| Correctness benefit | Handles UNC paths, PATHEXT quoting, symlinks on Windows |
| Dependency weight | `which` adds 3 transitive dependencies |
| Ecosystem standing | Top-500 most downloaded crates on crates.io |
