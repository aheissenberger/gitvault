# REQ-109: Contextual Directory Creation Utility

**ID:** S-20260304-R0109
**Status:** Partial (utility implemented; selective adoption)
**Priority:** Low
**Category:** Engineering / Diagnostics

## Problem

`std::fs::create_dir_all` appears ~78 times in the codebase. Each call produces a generic
error with no path context, making failures difficult to diagnose. A centralized helper
ensures consistent, debuggable error messages across the codebase.

## Requirement

A `fs_util::ensure_dir(path: &Path) -> Result<(), GitvaultError>` utility SHALL provide
a canonical wrapper that includes the directory path in the error message. New call sites
MUST use `fs_util::ensure_dir`. Existing call sites SHOULD be migrated incrementally.

## Acceptance Criteria

- AC1: `fs_util::ensure_dir` exists and includes the path in any error message.
- AC2: New code uses `fs_util::ensure_dir`.
- AC3: `src/commands/init.rs` and `src/ssm/refs.rs` are migrated.

## Implementation

- `src/fs_util.rs`: `pub fn ensure_dir`
- `src/commands/init.rs`: migrated
- `src/ssm/refs.rs`: migrated
