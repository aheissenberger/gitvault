# REQ-113: Breaking Changes to `encrypt`

**ID:** S-20260304-R113
**Status:** Proposed
**Priority:** High
**Category:** Breaking Change / DX Coherence

## Problem

`gitvault encrypt` currently accepts three flags that each silently change the output model:

| Flag | Effect | Problem |
|---|---|---|
| *(none)* | Whole file ŌåÆ `store/<env>/<flat-name>.age` | Loses source directory structure |
| `--keep-path` | Whole file ŌåÆ `store/<env>/<mirrored-path>.age` | Needed to fix confusing default |
| `--fields` | In-place field write | Completely different output location |
| `--value-only` | In-place .env value write | Completely different output location |

The result: one command, four different output behaviours depending on flag combination.
`--keep-path` exists solely to paper over a confusing default. `--fields` and `--value-only`
have been moved to the new `seal` command (REQ-112), eliminating two of the four modes.
This spec cleans up the remaining inconsistency by making path-mirroring unconditional.

## Breaking Changes

### 1. Remove `--fields`

`encrypt --fields` is superseded by `seal --fields` (REQ-112). Callers must migrate:

```bash
# Before
gitvault encrypt config/app.json --fields db.password,api.key --env prod

# After
gitvault seal config/app.json --fields db.password,api.key
```

### 2. Remove `--value-only`

`encrypt --value-only` is superseded by `seal` on `.env` files (REQ-112):

```bash
# Before
gitvault encrypt .env --value-only --env dev

# After
gitvault seal .env
```

### 3. Remove `--keep-path` ŌĆö path mirroring becomes unconditional

The flat output layout was the original default but was immediately problematic:

```
# Before (flat ŌĆö default, confusing)
gitvault encrypt services/auth/config.json --env prod
ŌåÆ .gitvault/store/prod/config.json.age   ŌåÉ directory structure lost

# Before (mirrored ŌĆö opt-in, correct)
gitvault encrypt services/auth/config.json --keep-path --env prod
ŌåÆ .gitvault/store/prod/services/auth/config.json.age

# After (mirrored ŌĆö unconditional, no flag)
gitvault encrypt services/auth/config.json --env prod
ŌåÆ .gitvault/store/prod/services/auth/config.json.age
```

The mirrored layout means:
- The store path is always mechanically derivable from the source path and env.
- `decrypt` can resolve the store path from the source path (REQ-114).
- `status` can match source files to store artifacts without heuristics.
- Monorepos with nested config directories work correctly without extra flags.

## `encrypt` Command After Changes

```
gitvault encrypt [OPTIONS] <FILE>
```

| Flag | Default | Description |
|---|---|---|
| `<FILE>` | ŌĆö | File to encrypt (must not be a `.age` file) |
| `-r, --recipient <PUBKEY>` | `.gitvault/recipients/` | age public key (repeat for multi-recipient) |
| `-e, --env <ENV>` | ŌĆö | Environment (overrides `GITVAULT_ENV`) |
| `-i, --identity <IDENTITY>` | ŌĆö | Identity key file |

Output: `.gitvault/store/<env>/<relative-source-path>.age`

**Removed flags:** `--fields`, `--value-only`, `--keep-path`

## Migration for Existing Repos

Repos using the flat layout (no `--keep-path`) will have store files at incorrect paths after
this change. The `gitvault migrate-store` command handles this:

```bash
gitvault migrate-store --dry-run   # show what would move
gitvault migrate-store             # move flat files to mirrored paths
```

It scans `.gitvault/store/<env>/` for `.age` files that do not correspond to a mirrored
source path and attempts to match them to source files by filename. Ambiguous cases
(multiple source files with the same name) are reported and left for manual resolution.

## Implementation Notes

- `compute_output_path` in `src/commands/encrypt.rs`: remove the `keep_path: bool` parameter;
  always use the mirrored-path branch (what was `if keep_path { ... }`).
- `cli.rs` `Encrypt` variant: remove `keep_path`, `fields`, `value_only` fields.
- `dispatch.rs`: remove corresponding arguments from `cmd_encrypt` call.
- `cmd_encrypt`: remove `--fields` branch, `--value-only` branch, `keep_path` parameter.
- All test assertions referencing flat store paths must be updated to mirrored paths.
