## Overview
The plaintext returned by `crypto::decrypt()` and `crypto::decrypt_stream()` must be zeroized when no longer needed to prevent sensitive data from lingering in memory. This spec wraps those return types in `Zeroizing<Vec<u8>>` so the operating system zeros the memory when the value is dropped.

## Key Changes

### crypto::decrypt Type Signature
- **Before:** `pub fn decrypt(...) -> Result<Vec<u8>, GitvaultError>`
- **After:** `pub fn decrypt(...) -> Result<Zeroizing<Vec<u8>>, GitvaultError>`

### Caller Updates
All callers in `src/repo/paths.rs`, `src/commands/recipients.rs`, and other modules must deref or unwrap the `Zeroizing<Vec<u8>>` value when passing it to downstream functions:
```rust
let plaintext = crypto::decrypt(...)?;  // Zeroizing<Vec<u8>>
let data = &*plaintext;  // Deref to &[u8]
process_data(data)?;
// plaintext is dropped here and zeroized
```

### FileOutcome::Rekeyed Enum Variant
The `FileOutcome::Rekeyed` variant that holds cached plaintext during rotation now uses `Zeroizing<Vec<u8>>` so the memory is zeroized when the enum is dropped at the end of the rotation command.

## Risk & Mitigation
- **Mechanical refactoring:** No new logic paths, only type wrapping.
- **Test coverage:** All decryption tests continue to pass; behavior is identical.
