## Overview
The HMAC token key is a sensitive 32-byte value stored on disk as a hex string. An attacker with write access to the key file could corrupt it to all zeros, creating a degenerate key. An all-zero HMAC key is cryptographically worthless and would likely result in invalid signatures. This spec validates that the loaded key has at least some non-zero bytes before it is used.

## Key Changes

### Entropy Check in load_or_create_token_key
```rust
match fs::read_to_string(&key_path) {
    Ok(content) => {
        let bytes = hex::decode(content.trim())?;
        
        // NEW: Check for all-zero entropy
        if bytes.iter().all(|&b| b == 0) {
            return Err(GitvaultError::Other(
                "token key is all-zeros — possible file tampering".to_string()
            ));
        }
        
        // Proceed with key usage
        let key = /* copy to [u8; 32] */;
        Ok(key)
    }
    // ... rest of function ...
}
```

## Testing
- **Unit test:** Write a key file containing all zeros (as hex: "0000...0000"), attempt to load it, and verify the error is returned.
- **Integration test:** Verify that a corrupted key file causes the barrier to reject tokens until the key is regenerated.

## Risk & Mitigation
- **False positives:** Extremely unlikely; the probability of 32 random bytes being all zero is 2^-256.
- **Error message:** Clear and actionable; prompts investigation of file integrity.

## Note
This check is a defense-in-depth measure. It is not a substitute for proper file permissions and access controls on the key file.
