## Overview
The current implementation uses an `unsafe extern "C"` binding to the POSIX `isatty()` function to detect whether stdin is a terminal. Rust's standard library provides a safer, platform-abstracted alternative: `std::io::IsTerminal::is_terminal()`. This spec replaces the unsafe code with the standard library trait method.

## Key Changes

### Remove Unsafe FFI Block
```rust
// Before
#[cfg(unix)]
mod libc_isatty {
    use std::os::unix::io::AsRawFd;
    extern "C" {
        fn isatty(fd: libc::c_int) -> libc::c_int;
    }
    
    pub fn is_stdin_tty() -> bool {
        unsafe { isatty(std::io::stdin().as_raw_fd()) != 0 }
    }
}

#[cfg(not(unix))]
mod libc_isatty {
    pub fn is_stdin_tty() -> bool {
        false
    }
}
```

### Replace with Standard Library
```rust
// After
use std::io::IsTerminal;

fn is_stdin_tty() -> bool {
    std::io::stdin().is_terminal()
}
```

The `IsTerminal` trait is implemented for `Stdin` on all platforms (Unix, Windows, etc.) and handles platform differences internally.

## Affected Code
In `init_identity_from_stdin`, the TTY check is updated:
```rust
if is_stdin_tty() {
    // Interactive TTY: prompt for input
} else {
    // Piped input: read from stdin
}
```

## MSRV Compatibility
`IsTerminal` was stabilized in Rust 1.70. The project's MSRV is 1.93, so this change is compatible and safe.

## Testing
- **Unit tests:** Verify that TTY detection works correctly in both interactive and piped scenarios.
- **Integration tests:** Confirm that identity initialization succeeds with and without a terminal.

## Risk & Mitigation
- **Behavioral impact:** None; `is_terminal()` is a direct replacement for the unsafe `isatty()` call.
- **Platform compatibility:** The trait is portable; no platform-specific tests needed.
- **Security gain:** Elimination of unsafe code reduces attack surface.
