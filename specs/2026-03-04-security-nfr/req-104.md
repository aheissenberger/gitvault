## Problem

`GITVAULT_IDENTITY_FD` and `GITVAULT_IDENTITY_PASSPHRASE_FD` are read from the environment as
strings and parsed with `.parse::<u32>()`. The parsed `u32` is then cast to `i32` and passed to
`File::from_raw_fd(fd as i32)`.

If the `u32` value is greater than `i32::MAX` (2,147,483,647), the cast wraps to a negative
integer. On all POSIX platforms, negative FD values are invalid. Passing them to
`File::from_raw_fd` is **undefined behavior** in Rust's memory model: the safety contract of
`from_raw_fd` requires a valid, open file descriptor.

An attacker who can control environment variables (e.g. via a compromised parent process or
a `.env` file injection) could supply `GITVAULT_IDENTITY_FD=4294967295` and trigger this path.

## Solution

Add an explicit range check before each `from_raw_fd` call:

```rust
if fd > i32::MAX as u32 {
    return Err(GitvaultError::Usage(format!(
        "GITVAULT_IDENTITY_FD {fd}: file descriptor value exceeds i32::MAX"
    )));
}
```

Update the `// SAFETY:` comment to document that the check has been performed.

## Implementation

Both `load_identity_from_fd` and `load_passphrase_from_fd` in `src/identity.rs` were updated
to check `fd > i32::MAX as u32` before the `unsafe { File::from_raw_fd(fd as i32) }` call.
