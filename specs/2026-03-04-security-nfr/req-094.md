## Overview
The `compute_hmac` function in `src/barrier.rs` currently uses `HmacSha256::new_from_slice()`, which accepts a slice and returns a `Result` that requires runtime error handling via `expect()`. The `hmac` crate also provides a typed API using `GenericArray` that enforces the correct key size (32 bytes for SHA256) at compile time. This spec replaces the slice-based API with the typed API to eliminate the runtime error check.

## Key Changes

### Before
```rust
pub fn compute_hmac(key: &[u8; 32], input: &[u8]) -> [u8; 32] {
    let mut mac = HmacSha256::new_from_slice(key)
        .expect("HMAC key size is always 32 bytes");
    mac.update(input);
    
    let result = mac.finalize();
    let code_bytes = result.into_bytes();
    // Convert GenericArray to [u8; 32]
    let mut output = [0u8; 32];
    output.copy_from_slice(&code_bytes);
    output
}
```

### After
```rust
pub fn compute_hmac(key: &[u8; 32], input: &[u8]) -> [u8; 32] {
    let key_array: GenericArray<u8, U32> = (*key).into();
    let mut mac = HmacSha256::new(key_array);
    mac.update(input);
    
    let result = mac.finalize();
    let code_bytes = result.into_bytes();
    let mut output = [0u8; 32];
    output.copy_from_slice(&code_bytes);
    output
}
```

The `HmacSha256::new()` constructor takes a `GenericArray<u8, U32>` (32 bytes), enforcing the size at compile time. No `expect()` or runtime error handling is needed.

## Type Conversions
- `&[u8; 32]` → `GenericArray<u8, U32>` via `.into()` or `From` trait.
- Return type of `finalize()` is a `GenericArray<u8, U32>`, which is then converted to `[u8; 32]` via `copy_from_slice()`.

## Verification
- **Compile-time check:** The code will not compile if the key type is incorrect; the type mismatch is caught immediately.
- **No behavioral change:** The HMAC computation and output are identical.
- **Zero runtime overhead:** The conversion from `[u8; 32]` to `GenericArray` is a zero-copy, compile-time abstraction.
