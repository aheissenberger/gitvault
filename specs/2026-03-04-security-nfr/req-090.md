## Overview
When gitvault spawns a `git` subprocess to read user metadata, repository state, or configuration, it inherits environment variables from the parent process. A malicious actor could set `GIT_DIR`, `GIT_CONFIG`, or `GIT_CONFIG_GLOBAL` to inject arbitrary behavior. This spec requires that all such subprocesses have those variables explicitly removed before spawning.

## Key Changes

### Helper Function: sanitize_git_cmd()
```rust
fn sanitize_git_cmd(cmd: &mut Command) {
    cmd.env_remove("GIT_DIR")
       .env_remove("GIT_CONFIG")
       .env_remove("GIT_CONFIG_GLOBAL");
}
```

### Apply to All Metadata Reads
Every `Command::new("git")` call that reads metadata in production code must call `sanitize_git_cmd()`:

- `src/commands/recipients.rs`: `git config user.name`
- `src/identity.rs`: `git config user.email`, etc.
- `src/repo/drift.rs`: `git log --diff-filter=...` (history scan)
- `src/commands/admin.rs`: Any git subcommand for reading state
- `src/commands/init.rs`: Initial repository setup git calls
- `src/ssm/mod.rs`, `src/ssm/refs.rs`: SSM integration with git

### Exception: Repository Discovery
Calls like `git rev-parse --show-toplevel` that are specifically intended to find the repository root may keep inherited `GIT_DIR` (if set). These are rare and should be explicitly documented with a comment:
```rust
// EXCEPTION: git rev-parse --show-toplevel needs GIT_DIR to discover the repo
let mut cmd = Command::new("git");
cmd.arg("rev-parse").arg("--show-toplevel");
// DO NOT sanitize; this call intentionally uses GIT_DIR
```

## Audit & Testing
1. **Grep audit:** `grep -rn "Command::new(\"git\")" src/` to find all git subprocesses.
2. **Code review:** Verify each call is either sanitized or explicitly documented as an exception.
3. **Unit tests:** Confirm that sanitized calls do not pick up malicious `GIT_DIR` or `GIT_CONFIG` from the environment.

## Risk & Mitigation
- **Behavioral impact:** None under normal operation; only affects behavior when malicious env vars are set.
- **Cross-platform:** Works on Linux, macOS, and Windows (env var removal is portable).
