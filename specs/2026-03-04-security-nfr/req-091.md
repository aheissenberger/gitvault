## Overview
Pre-checking file existence with `path.exists()` before reading introduces a TOCTTOU (time-of-check-to-time-of-use) race: the file could be deleted or replaced between the check and the read, causing confusion about the error. The robust pattern is to attempt the operation and handle the "not found" error directly.

## Key Changes

### Before
```rust
if key_path.exists() {
    let content = fs::read_to_string(&key_path)?;
    // ...decode and validate...
    Ok(key)
} else {
    // ...create new key...
    Ok(new_key)
}
```

### After
```rust
match fs::read_to_string(&key_path) {
    Ok(content) => {
        // ...decode and validate...
        Ok(key)
    }
    Err(e) if e.kind() == io::ErrorKind::NotFound => {
        // ...create new key...
        Ok(new_key)
    }
    Err(e) => Err(GitvaultError::Io(e)),
}
```

## Behavior
- If the key file exists and is readable, it is loaded, decoded, validated, and returned.
- If the key file does not exist, a new key is generated, encoded as hex, written to disk, and returned.
- If any other I/O error occurs (permission denied, disk full, etc.), it is propagated to the caller as `GitvaultError::Io`.

## Risk & Mitigation
- **Behavioral change:** None; the function's external behavior is identical.
- **Race elimination:** Eliminates a narrow TOCTTOU window.
- **Error handling:** More precise error classification (permission issues are distinct from missing files).
